## 一、项目愿景

对标 Tooltrace，打造一款**个人使用**的网页端智能工具收纳嵌件生成工具。

**双一等模式**：Foam（Kaizen/Shadowbox 泡沫，CNC/激光切割）与 Gridfinity（42mm 网格 3D 打印）并列，切换时工具轨迹保留不丢失。

**核心流程**：上传工具照片 → AI 自动识别轮廓（可手动描边修正）→ 网页端可视化编辑（移动、旋转、调整抓手/对称）→ 本地历史画廊随时回看 → 导出工业级文件（STL/STEP/3MF/DXF/SVG/PDF）。

**核心原则**：前端（JS）负责交互与预览，后端（Python）负责核心算法与 CAD 运算。采用 Tauri/Electron 桌面打包，数据本地持久化（**无云端账户/社区**，画廊为个人本地历史）。

> 📌 **范围声明（个人使用）**：完整对标 Tooltrace 的*功能能力*——双模式、手动描边修正、自定义形状、精度档、强制对称、导入已有工具、PDF 校对、本地画廊历史。
> **不含** Tooltrace 的 SaaS 社交/商业化层：云端账户、社区发布/评分、分享短链、Try-Before-You-Buy 付费墙、5S/精益/FOD 的 B2B 合规销售话术（个人使用不需要，但"缺件一眼可见"等可视化收益由嵌件本身自带）。

## 三、技术架构与开发需求（Technical Spec）

### 1. 总体架构（Architecture）

采用 **前后端分离 + WebSocket** 架构。

- **前端（Browser）**：JavaScript（Vue/React）+ Three.js + TailwindCSS
- **后端（Server）**：Python + FastAPI + OpenCV + PythonOCC（OCCT）
- **通信**：HTTP（上传/下载）+ WebSocket（实时参数调整/重算）

### 2. 后端开发需求（Python Backend）

职责：承载所有 CPU 密集型与 CAD 运算，不负责渲染。

| ID | 模块 | 开发需求 | 现有技术 | 需开发 |
|----|------|----------|----------|--------|
| B1 | API 服务 | FastAPI 搭建 RESTful API 与 WebSocket 接口。 | - | 新建 |
| B2 | 算法引擎 | 封装现有的 SAM、透视校正、轮廓提取代码。 | 已有 | 封装 |

### 4. 核心数据流转（Data Flow）

以「拖拽手指槽」为例：

1. **前端**：用户拖拽手指槽 3D 对象 → 释放鼠标 → 前端计算新的 (x, y) 坐标。
2. **前端**：通过 WebSocket 发送 `{ type: "update_slot", toolId: 1, x: 10, y: 20 }`。
3. **后端**：WS 服务接收参数 → 更新内存中 Tool 对象的 slot_position。
4. **后端**：CAD 引擎重新计算：`Plate - ToolCut - SlotCut` → 生成新的 STL 二进制流。
5. **后端**：WS 发送 `{ type: "new_model", format: "stl", data: <binary> }`。
6. **前端**：接收二进制流 → 销毁旧 Mesh → 加载新 STL Mesh → 场景更新。

## 四、开发阶段划分（Roadmap）

### Phase 1: MVP（最小可行性产品）— 2 Weeks
目标：能在网页上看到单工具的嵌件，并能调整深度。
- [ ] 后端：FastAPI 封装现有算法，提供 `/upload` 和 `/generate` 接口。
- [ ] 后端：实现单工具 Extrude 生成 STL。
- [ ] 前端：上传图片界面，Three.js 显示生成的 STL。
- [ ] 前端：滑块控件调整 `embed_depth`，点击按钮触发重算。

### Phase 2: 交互与排版 — 3 Weeks
目标：支持多工具排版、手指槽拖拽、双模式与本地画廊。
- [ ] 后端：集成 rectpack 实现自动排版。
- [ ] 后端：实现手指槽生成逻辑（Offset + Boolean Cut）。
- [ ] 后端：实现 Foam / Gridfinity **模式引擎**（B7），切换保轨迹。
- [ ] 前端：实现多工具列表显示。
- [ ] 前端：实现工具拖拽（DragControls）和旋转按钮、强制对称（F3.11）。
- [ ] 前端：实现手指槽拖拽，并绑定 WebSocket 实时更新。
- [ ] 前端 + 后端：实现**本地画廊 / 历史**（F1.6 / B8 / F7），保存与回看。

### Phase 3: 专业功能与导出 — 2 Weeks
目标：达到 Tooltrace 专业级体验（含手动修正与全格式）。
- [ ] 后端：接入 PythonOCC，实现 STEP 导出。
- [ ] 后端：实现 Gridfinity 底板生成逻辑（磁吸孔）。
- [ ] 前端：添加公差预设、拔模角度、间距等高级参数面板。
- [ ] 前端：实现 2D 基元轮廓叠加显示（Debug/校对用）。
- [ ] 前端 + 后端：实现**手动描边修正**（F2.5 控制点）、**自定义形状绘制**（F2.6）、**Detail 精度档**（F2.7）、**导入已有工具**（F3.12）。
- [ ] 导出补齐：**PDF 1:1 校对图**（F5.5）。
