# 📐 工具轮廓扫描 & 3D 收纳生成器 — 产品需求文档（PRD）

## 基本信息

- **平台**：Web 浏览器（Chrome 优先）
- **核心技术**：OpenCV.js + SAM（Segment Anything）+ OpenCASCADE WASM

## 一、产品概述

### 1.1 一句话定位

用户将工具平放在 A4/Letter 纸上俯拍上传，系统自动识别纸张标定尺寸、AI 提取工具轮廓、配置嵌件参数，一键导出 STL / STEP / DXF / SVG，用于 3D 打印或激光切割工具收纳嵌件。

### 1.2 目标用户

- 3D 打印爱好者 / Maker
- 工厂车间工具管理（5S / Kaizen 泡沫嵌件）
- Gridfinity 收纳系统用户

### 1.3 核心价值

从"拍照"到"可打印模型"，全程浏览器完成，无需安装软件、无需上传云端，保护用户隐私。

## 二、用户流程（6 步向导）

```
Step 1: 上传照片 ──→ Step 2: 纸张检测与标定 ──→ Step 3: AI 工具轮廓提取
                      │
Step 6: 下载文件 ←── Step 5: 导出预览 ←── Step 4: 轮廓编辑 & 嵌件参数配置
```

## 三、功能需求详述

### 3.1 图像采集规范（引导页）

| 要求     | 说明                                          |
| -------- | --------------------------------------------- |
| 背景     | 白色/浅色 A4 或 Letter 纸                     |
| 拍摄角度 | 尽量正俯视，距离远 + 光学变焦（减少透视畸变） |
| 光照     | 均匀漫射光，避免工具投下浓重阴影              |
| 工具摆放 | 工具间留间距 ≥ 5mm，不重叠                    |
| 支持格式 | JPG / PNG / WebP                              |

📌 关键提示：引导页需告知用户"远距离 + 变焦"拍摄，避免视差误差（parallax error）。

### 3.2 纸张检测与透视校正

| 功能                   | 技术方案                                                    |
| ---------------------- | ----------------------------------------------------------- |
| 自动检测纸张四角       | OpenCV.js 边缘检测 + 四边形拟合                             |
| 透视校正（Homography） | 将歪斜的纸张 warp 为正射俯视图                              |
| 尺寸标定               | 已知 A4=210×297mm 或 Letter=215.9×279.4mm，计算像素/mm 比例 |
| 手动修正               | 若自动检测失败，用户可手动拖拽四角                          |
| 纸张格式选择           | 下拉选择 A4 / A5 / Letter，默认 A4                          |

### 3.3 AI 工具轮廓提取

| 功能         | 技术方案                                                             |
| ------------ | -------------------------------------------------------------------- |
| 一键 AI 分割 | 集成 SAM（Segment Anything）ONNX 模型，浏览器 WASM + Web Worker 运行 |
| 交互方式     | 点击工具区域 → 实时生成掩码 → 提取外轮廓                             |
| 多工具支持   | 每张照片可添加多个工具，逐个点击识别                                 |
| 备选方案     | 传统 CV 方案（Canny + 形态学 + findContours）作为轻量 fallback       |
| 模型缓存     | SAM 权重首次下载后缓存至 IndexedDB / OPFS，约 40-100MB               |

### 3.4 轮廓编辑器

| 功能           | 说明                                    |
| -------------- | --------------------------------------- |
| 矢量预览       | SVG Canvas 展示轮廓路径（毫米坐标）     |
| 拖点微调       | 拖动轮廓顶点修正 AI 误差                |
| 增/删点        | Ctrl+点击加点的，Ctrl+点击点删除        |
| 平滑处理       | 高斯/Chaikin 平滑去噪                   |
| 旋转           | 拖拽旋转手柄调整工具方向                |
| 复制/删除      | 多工具管理                              |
| 偏移（Offset） | 轮廓向内/外偏移 0~2mm，用于补偿打印公差 |

### 3.5 嵌件参数配置

| 参数            | 默认值              | 说明                                      |
| --------------- | ------------------- | ----------------------------------------- |
| 轮廓偏移        | +0.3mm（外扩）      | 补偿打印机公差，确保工具能放入            |
| 底板厚度        | 2mm                 | 嵌件底部支撑厚度                          |
| 腔体深度        | 15mm                | 工具嵌入深度                              |
| 手指取放槽      | 自动/手动           | 在轮廓边缘凹处生成半圆/椭圆凹槽，深度可设 |
| 布局模式        | 单工具 / 多工具同板 | 多工具时自动排列打包                      |
| Gridfinity 模式 | 可选                | 生成标准 42mm Gridfinity 底座网格         |

### 3.6 3D 模型生成与预览

| 功能     | 技术方案                                                      |
| -------- | ------------------------------------------------------------- |
| 矢量→3D  | 闭合多边形拉伸（Extrude）生成实体                             |
| 布尔运算 | 腔体挖空 + 手指槽切割                                         |
| 几何内核 | OpenCASCADE WASM（B-Rep 实体建模，支持 STEP 导出）或 Manifold |
| 3D 预览  | Three.js 渲染，支持旋转/缩放                                  |
| 尺寸标注 | 显示工具轮廓的实际毫米尺寸                                    |

### 3.7 导出格式

| 格式      | 用途                                                |
| --------- | --------------------------------------------------- |
| STL       | 直接切片 3D 打印（最常用）                          |
| STEP      | 导入 Fusion 360 / SolidWorks 二次编辑（B-Rep 保留） |
| 3MF       | 多色/多 Body 打印                                   |
| DXF / SVG | 激光切割泡沫或亚克力衬板                            |

## 七、关键风险与应对

| 风险                                 | 应对策略                                               |
| ------------------------------------ | ------------------------------------------------------ |
| SAM 模型体积大（~100MB），首次加载慢 | 分模型档位（Tiny/Small/Base），按需下载；OPFS 持久缓存 |
| 手机广角拍摄透视畸变严重             | 引导页强调"远距离+变焦"；提供 ArUco 角标增强标定方案   |
| 复杂工具（钳子/扳手）投影轮廓不完整  | 允许用户手动绘制补充轮廓；提供"多层嵌件"选项           |
| WASM 内存限制                        | 大模型推理放 Web Worker；复杂场景分步处理              |

# 🛠️ Web 端「工具轮廓扫描 & 收纳生成器」开发方案（前端先行版）

## 一、总体开发策略（重点）

- ✅ 前端优先：先不管算法，先用假数据把 UI/UX 跑通
- ✅ 模块化：图像 / 矢量 / 3D 各自独立，后期替换实现
- ✅ 渐进增强：每一步都能打开浏览器看到效果
- ✅ 本地优先：所有重计算放 Web Worker，UI 不卡顿

| 层级    | 技术                                      |

```ts
type Step =
  | "upload"
  | "calibration"
  | "segmentation"
  | "editor"
  | "params"
  | "export";

interface AppState {
  step: Step;
  setStep: (step: Step) => void;

  imageUrl: string | null;
  setImageUrl: (url: string) => void;

  contours: any[]; // 后期替换为真实轮廓类型

export const useStore = create<AppState>((set) => ({
  step: "upload",
  setStep: (step) => set({ step }),

  imageUrl: null,
  setImageUrl: (imageUrl) => set({ imageUrl }),

  contours: [],
  setContours: (contours) => set({ contours }),
}));
```

## 八、Step 3：图片上传页

src/features/upload/UploadPage.tsx

```tsx
import { useStore } from "@/app/store";

export function UploadPage() {
  const setImageUrl = useStore((s) => s.setImageUrl);
  const setStep = useStore((s) => s.setStep);

  const onFile = (e: React.ChangeEvent<HTMLInputElement>) => {
    const file = e.target.files?.[0];
    if (!file) return;
    setImageUrl(URL.createObjectURL(file));
    setStep("editor"); // 暂时跳到 editor
  };

  return (
    <div className="p-8">
      <h2 className="text-xl mb-4">上传工具照片</h2>
      <input type="file" accept="image/*" onChange={onFile} />
    </div>
  );
}
```

## 九、Step 4：Fabric.js 轮廓编辑器（核心）

src/components/CanvasEditor.tsx

```tsx
import { useEffect, useRef } from "react";
import { fabric } from "fabric";
import { useStore } from "@/app/store";

export function CanvasEditor() {
  const canvasRef = useRef<HTMLCanvasElement>(null);
  const imageUrl = useStore((s) => s.imageUrl);
  const setContours = useStore((s) => s.setContours);

  useEffect(() => {
    if (!canvasRef.current || !imageUrl) return;

    const canvas = new fabric.Canvas(canvasRef.current, {
      width: 800,
      height: 600,
      backgroundColor: "#eee",
    });

    // 加载图片
    fabric.Image.fromURL(imageUrl, (img) => {
      img.scaleToWidth(800);
      canvas.add(img);

      // ✅ 假轮廓（模拟 AI 结果）
      const fakePath = new fabric.Path(
        "M 200 150 L 400 150 L 450 300 L 150 300 Z",
        {
          fill: "rgba(0,150,255,0.3)",
          stroke: "blue",
          strokeWidth: 2,
          selectable: true,
        },
      );

      canvas.add(fakePath);
      setContours([fakePath]);
    });

    return () => canvas.dispose();
  }, [imageUrl]);

  return <canvas ref={canvasRef} />;
}
```

## 十四、依赖安全问题及解决方案

在安装依赖时出现以下警告：

```
npm warn deprecated inflight@1.0.6: This module is not supported, and leaks memory.
npm warn deprecated npmlog@5.0.1: This package is no longer supported.
npm warn deprecated rimraf@3.0.2: Rimraf versions prior to v4 are no longer supported.
npm warn deprecated glob@7.2.3: Contains security vulnerabilities.
npm warn deprecated are-we-there-yet@2.0.0: This package is no longer supported.
npm warn deprecated gauge@3.0.2: This package is no longer supported.
npm warn deprecated tar@6.2.1: Contains security vulnerabilities.
```

### 解决方案

1. **定期依赖更新**：
   - 每月执行 `npm outdated` 检查过时依赖
   - 每季度执行 `npm audit` 安全审计
   - 优先更新有安全漏洞的依赖包

2. **依赖升级策略**：
   - 对于主版本号变更的依赖，先在开发分支测试
   - 使用 `npm audit fix` 修复可自动解决的安全问题
   - 对于无法自动修复的问题，寻找替代方案

3. **长期维护计划**：
   - 建立依赖更新日志
   - 设置自动化依赖检查（如 Dependabot）
   - 制定紧急安全补丁响应流程

## 十六、风险评估与应对策略

### 技术风险

1. **WASM 内存限制**：
   - 应对：大模型推理放 Web Worker，复杂场景分步处理

2. **SAM 模型体积大（~100MB）**：
   - 应对：分模型档位（Tiny/Small/Base），按需下载；OPFS 持久缓存

3. **手机广角拍摄透视畸变严重**：
   - 应对：引导页强调"远距离+变焦"；提供 ArUco 角标增强标定方案

### 性能风险

1. **复杂工具（钳子/扳手）投影轮廓不完整**：
   - 应对：允许用户手动绘制补充轮廓；提供"多层嵌件"选项
